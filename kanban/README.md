# Piclaw Kanban Board

This board is the **project-local kanban** for piclaw.

## What belongs here

- piclaw feature work
- piclaw bugs and regressions
- piclaw architecture, tooling, release, and UX tickets

## What does not belong here

- tickets explicitly tagged `non-piclaw`
- unrelated infrastructure or side-project ideas
- new-project exploration that should live outside the piclaw repo

## Notes

- This board mirrors the standard lane layout used elsewhere.
- Prefer keeping piclaw work here instead of in a shared global board.
- Keep non-piclaw tickets in the global `/workspace/kanban` board unless the user requests a different home.

## Layout

- `00-inbox/`
- `10-next/`
- `20-doing/`
- `30-blocked/`
- `40-review/`
- `50-done/`
- `_templates/`
